Brian showed me some cool ssh things today.  First off, we used ssh keys to allow password free logins, which I have done before.  Here is the old song and dance:

# if you want to use a different key pair first generate one
ssh-keygen -t rsa -f ~/.ssh/xterra-id_rsa
cd ~
cat .ssh/xterra-id_rsa.pub | ssh remote.host 'cat - >> .ssh/authorized_keys'

Then he showed me some info on the ~/.ssh/config file.  Setting up this file can make working with ssh among different hosts and different key pairs much easier.  Here is an example of what is in mine now:

Host release
 HostName xterra.bigtribe.com
 User release
 ForwardX11 no
 ForwardAgent no
 Cipher blowfish
 Compression yes
 IdentitiesOnly yes
 IdentityFile ~/.ssh/xterra-id_rsa

Host qaremote
 HostName g2.bigtribe.com
 User ageo
 ForwardX11 no
 ForwardAgent no
 Cipher blowfish
 Compression yes
 IdentitiesOnly yes
 IdentityFile ~/.ssh/qa-id_rsa

Host qamain
